# Reindex Procedure — Post OpenAI→Gemini Embedding Migration

## Prerequisite

Rotate `GOOGLE_API_KEY` in Railway env vars first.  
**Do NOT run this until you have a working key.**

Verify key works:
```bash
curl -s "https://generativelanguage.googleapis.com/v1beta/models/gemini-embedding-001?key=$GOOGLE_API_KEY" \
  | python3 -m json.tool | grep name
```
Expected: `"name": "models/gemini-embedding-001"`. If 403/404 → key not valid yet.

## Why reindex is needed

Existing vectors in `content_embeddings` were generated by OpenAI `text-embedding-3-small`.
New queries use Gemini `gemini-embedding-001`. These models live in different vector spaces —
cosine similarity between old docs and new queries will produce random noise results.

## Step 1 — Quick validation (no reindex yet)

```bash
cd ~/Clonnect/backend
set -a && source .env && set +a

.venv/bin/python3.11 -c "
from core.embeddings import generate_embedding
out = generate_embedding('Hola, soy Iris y enseño Pilates en Barcelona')
assert out and len(out) == 1536, f'FAIL: dim={len(out) if out else 0}'
print(f'OK dim={len(out)}, first 5: {[round(v,4) for v in out[:5]]}')
"
```

## Step 2 — Reindex content_embeddings

```bash
cd ~/Clonnect/backend
set -a && source .env && set +a

# Reindex all chunks for all creators (uses create_proposition_chunks.py --force-reindex)
.venv/bin/python3.11 scripts/create_proposition_chunks.py \
  --creator-id iris_bertran \
  --force-reindex \
  --dry-run    # remove --dry-run when ready to write
```

If `--force-reindex` flag doesn't exist, use the direct DB approach:

```bash
.venv/bin/python3.11 -c "
import asyncio, os
from sqlalchemy import create_engine, text

eng = create_engine(os.environ['DATABASE_URL'])
with eng.connect() as c:
    result = c.execute(text('SELECT COUNT(*) FROM content_embeddings'))
    print(f'Chunks to reindex: {result.scalar()}')
"
```

Then trigger content refresh via the API:
```bash
curl -X POST https://www.clonnectapp.com/maintenance/reindex-embeddings \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"creator_id": "iris_bertran", "force": true}'
```

## Rollback

If results are catastrophically bad after reindex, the issue is NOT the model switch
(Gemini gemini-embedding-001 is higher quality than OpenAI text-embedding-3-small).
Check:
1. `outputDimensionality=1536` is being sent in the request
2. `taskType=RETRIEVAL_DOCUMENT` for stored docs, `RETRIEVAL_QUERY` for searches
3. API key has embeddings quota enabled (AI Studio → check quotas)

Do NOT revert to OpenAI. If Gemini is consistently worse, investigate task_type mismatch.
