## 5. DECISIONES TOMADAS

1. **Modelo confirmado:** `gemini-2.5-flash-lite` (coste $0.000226/msg, 604ms latencia media)

3. **Memory budget:** 3000 chars es el nuevo default. No hay evidencia de que más chars ayude sin tests adicionales

5. **Gold examples contaminados:** 11 purgados. Pipeline de purga disponible en `scripts/purge_contaminated_gold_examples.py`
6. **Anti-loop:** La comparación de full-string (en lugar de primeros 50 chars) es el threshold correcto para detectar loops reales sin falsos positivos
7. **RAG:** El bug UUID estaba activo desde el inicio. Los 93 chunks existentes nunca habían tenido embeddings. Ahora 107/107 embebidos

## 7. LECCIONES

1. **Los sistemas de "mejora" eran net-negative.** Best-of-N, Self-Consistency y Reflexion son técnicas diseñadas para modelos más débiles. Con Gemini 2.5, añadían ruido y latencia sin mejorar calidad. La simplificación (+19 pts en Clone Score) fue mayor que cualquier "mejora" individual añadida.

2. **El test set es imprescindible.** Sin las 20 conversaciones con ground truth, no habríamos sabido que el guardrail anti-loop fallaba en 5/20 casos ni que el bug del CoT afectaba a audios. El score de 17.1% del eval diario era ruidoso; el test set controlado da señal limpia.

3. **La varianza entre runs es información.** Std dev 29.7 en run 1 reveló el problema del guardrail. Std dev 12.9 después del fix confirmó que se resolvió. La seed fija (run 5) elimina la varianza del few-shot sampling para comparaciones futuras.

4. **Los bugs silenciosos duelen más.** El bug UUID en RAG llevaba semanas (posiblemente meses) activo — 93 chunks, 0 embeddings, sin error visible en logs. El fix RAG es probablemente el cambio con mayor impacto a largo plazo de esta sesión.

5. **El guardrail `[:50]` era una trampa.** Comparar sólo los primeros 50 caracteres para detectar loops garantiza falsos positivos en cualquier modelo que genere saludos o aperturas similares. La heurística correcta es comparación de string completo contra el último mensaje.

6. **Menos contexto ≠ mejor respuesta.** 1200 chars de memoria forzaba al modelo a truncar hechos relevantes del lead. Con 3000 chars, el contexto del lead llega completo. El impacto en el score es difícil de aislar pero el efecto anecdótico en conv_009 (92.4%) y conv_020 (95.9%) sugiere que la memoria de hechos específicos sí impacta.
